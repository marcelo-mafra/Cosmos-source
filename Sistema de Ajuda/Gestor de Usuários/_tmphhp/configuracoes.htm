Configurações
 Anterior Acima Seguinte
Para conseguir realizar um login, antes os usuários dos aplicativos da suíte Cosmos devem fornecer as seguintes informações de configuração do sistema:


Protocolo de conexão
O protocolo de conexão é o mecanismo que será utilizado para realizar a conexão e tráfego de dados entre a aplicação cliente e o servidor de dados. O servidor de dados é um aplicativo responsável por fornecer diversas informações solicitadas pelos usuários. É possível verificar se um servidor está sendo executado observando-se o seu ícone de notificação, exibido ao lado do relógio do sistema.
Servidores
São os computadores nos quais as aplicações servidoras de dados foram instaladas. As aplicações clientes usam esta informação para inicializar os servidores de dados.
Porta de conexão
Número da porta de comunicação do sistema a ser utilizada no tráfego de dados. Esta informação sempre deve ser fornecida, mas apenas é relevante se você utiliza o protocolo de conexão socket TCP/IP. Tipicamente, o valor desta informação é 211.


Como estas informações são requeridas para que uma conexão entre aplicações clientes e servidoras de dados seja efetuada com sucesso, os aplicativos Cosmos exibirão mensagens de erro caso o usuário não as tenham informado.
O utilitário Borland Socket Server, instalado com o pacote Cosmos, deve estar sendo executado se você preferir utilizar o protocolo de conexão socket TCP/IP.
Se você preferir usar o protocolo DCOM e o seu servidor de dados estiver instalado em um computador diferente de uma aplicação cliente Cosmos, as configurações podem se tornar bem mais complexas. Isto ocorre devido ao fato de ser necessário configurar o serviço de DCOM do Microsoft Windows. Contudo, caso a aplicação servidora estiver instalada no mesmo computador de uma aplicação cliente, esta última configuração será desnecessária. Isto será válido mesmo que o banco de dados do Cosmos esteja localizado em um computador da sua rede.


Outras opções

A janela de configurações ainda oferece as seguintes opções:
Exibir login ao inicializar
Se marcada, indicará que a janela de login deverá ser exibida logo após a inicialização da aplicação Cosmos.
Usar balanceamento de carga
Se marcada, o sistema de conexão com a aplicação servidora irá determinar, dinamicamente, qual instância da aplicação servidora receberá as solicitações. Este recurso somente será usado caso você tenha indicado mais de um computador nos quais sua aplicação servidora foi instalada.
